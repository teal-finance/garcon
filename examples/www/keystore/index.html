Private key/value store

This Garcon example is a key/value store providing private storage for each client identified by its unique IP.

  • GET /keys lists the key names for each client (each IP). Only available when queried from local host 120.0.0.1.

  • POST /keys requires form body encoded with the application/x-www-form-urlencoded media type. The following command line sets the key AA: 

            curl -D - -X POST 127.0.0.1:8080/keys -d AA=12345
    The following command line gets the key AA: 
            curl -D - -X POST 127.0.0.1:8080/keys -d AA
    Multiple keys can be set and get simultaneously.

  • DELETE /keys accepts an optional form body encoded with the application/x-www-form-urlencoded media type. The following command line deletes all keys owned by the client: 

            curl -D - -X DELETE 127.0.0.1:8080/keys
    The following command line deletes only the key AA: 
            curl -D - -X DELETE 127.0.0.1:8080/keys -d AA
